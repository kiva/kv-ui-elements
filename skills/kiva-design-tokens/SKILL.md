# Kiva Design Tokens

Local snapshot of Kiva's design system variables, exported from Figma in W3C DTCG format and normalized per-collection.

## How to use

1. Start with `references/catalog.md` to see what collections exist, how many tokens each holds, and what modes are available.
2. Load the specific `references/tokens/<slug>.json` when you need values for a collection.
3. Each token carries: `type`, `values` (keyed by mode), `description`, `scopes` (Figma usage constraints), and `aliasOf` (where applicable).

## What's here

- **Primitives** live in `Global` — raw colors, type atoms, spacing, layout, radius, elevation atoms.
- **Semantic aliases** live in the `Alias - *` collections — `text/`, `background/`, `border/` etc., with theme or breakpoint mode variants.
- **Component mappings** live in `Mapped` — component-level bindings by component variant.

## Modes — how to read them

Modes mean different things in different collections. Don't assume "modes = themes."

| Collection | What the modes represent | Example |
|---|---|---|
| `Global`, `radius size`, `Alias - Border` | Single mode (`Mode 1`) — no variance | — |
| `Alias - Color` | **Theme variants** | `default`, `green-dark`, `green-light`, `marigold`, `stone` |
| `Alias - Text_Spacing_Layout` | **Breakpoints** | `desktop (lg)`, `desktop (xl)`, `mobile (sm)`, `mobile (xs)`, `tablet (md)` |
| `Alias - Elevation` | **Shadow attribute axes** (not themes — each mode is one dimension of a shadow) | `Blur`, `Opacity`, `Spread`, `y` |
| `Mapped` | **Component variants** (not themes) | `primary`, `secondary` |

**Theme switching** at runtime happens inside `Alias - Color` — pick the token (e.g. `background.action`), then read the value for the current theme mode. `Mapped` does not carry theme variance; it picks *which* semantic alias a given component variant binds to, and the alias itself carries the theme modes.

**Resolving a value for a themed component:** start at `Mapped`, pick the component variant mode, follow `aliasOf.<variant>` into the referenced semantic alias collection, then read the value for the current theme mode.

## Deprecated alias targets

Some tokens in `Mapped` reference `Alias - Color (deprecated)` — a collection intentionally not included in this snapshot. The current equivalent almost always lives in `Alias - Color` at the same leaf path (e.g. `default theme/background/action` in the deprecated collection ≈ `background.action` with the `default` theme mode in the current collection). Prefer the current collection for cross-theme values; treat the deprecated reference as a pointer to historical structure, not a source of truth.

## Alias paths use slashes, keys use dots

Inside `aliasOf`, the `name` field uses `/` as a path separator (Figma's native format). The token keys in `tokens` use `.` as the separator. To look up an alias target, replace `/` with `.` — e.g. alias name `color/eco-green/4` maps to key `color.eco-green.4`.

## Freshness

The snapshot is refreshed manually by running the `kiva-design-tokens-sync` skill against a new Figma export. Check `references/raw/MANIFEST.md` for the last export date.

## Do not

- Call Figma MCP tools from this skill. The snapshot is the source.
- Edit `references/tokens/*.json` or `references/catalog.md` by hand — they are generated by the sync.
